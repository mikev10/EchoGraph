# Task Files

This directory contains detailed task files for complex features.

## Structure

Each task file follows this naming convention:
```
TASK-XXX-feature-name.md
```

## Task File Template

```markdown
# [TASK-XXX] Feature Name

**Status**: Pending | In Progress | Completed
**Started**: YYYY-MM-DD
**Completed**: YYYY-MM-DD (when done)

## Context

Brief description of the feature and why it's needed.

## Subtasks

- [ ] Subtask 1
- [ ] Subtask 2
- [ ] Subtask 3

## Notes

Any relevant notes, decisions, or blockers.

## Testing

- [ ] Unit tests
- [ ] Integration tests
- [ ] Manual testing checklist

---
**Last Updated**: YYYY-MM-DD
```

## Usage

1. Create a new task file when starting a complex feature
2. Break down into subtasks
3. Update progress as you work
4. Move to completed when done
